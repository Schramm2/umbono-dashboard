# Umbono evidence shot list

Use demo mode for public captures unless a live provider run is explicitly labeled and cleared for publication. Never include credentials, private prompts, or fabricated live results.

## 1. Landing page

- Viewport: 1440 × 1000.
- Keep the complete hero value proposition, primary action, and deterministic ranking visual in frame.
- Capture both the dark system theme and one mobile layout.

## 2. Evaluation set and run controls

- Viewport: 1440 × 1000.
- Show the “Community guidance review” set, its selected prompt, and three synthetic model profiles.
- Keep Demo selected and the “No credentials” label visible.
- Capture before running so the visitor can see the complete input state.

## 3. Output comparison

- Run the selected prompt.
- Capture one selected response with its output tabs, latency, tokens, and illustrative cost visible.
- Keep the “Synthetic fixture” label in frame.

## 4. Human-defined weighted scoring

- Select one output and set the sliders to non-default values.
- Keep all criterion labels and weights visible.
- Capture the calculated score and the explanation that the result exists only in browser memory.

## 5. Leaderboard update

- Include the “Session update” marker on the newly scored model.
- Show overall score, values-alignment rate, p95 latency, cost per 1k tokens, token total, and run count.
- Capture the calculation-method note beneath the table.

## Recording option

A 30-45 second recording can follow the same sequence. Do not edit it in a way that implies provider calls or persistence while Demo mode is selected.
